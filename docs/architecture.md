## 1. 技术栈概览

```
┌─────────────────────────────────────────────────────────┐
│                    Claude Code CLI                       │
├─────────────────────────────────────────────────────────┤
│  运行时      │  Bun (单文件可执行)                        │
│  语言        │  TypeScript (strict mode)                 │
│  终端 UI     │  React + Ink (组件化 TUI)                  │
│  CLI 解析    │  Commander.js                             │
│  Schema 校验 │  Zod v4                                   │
│  代码搜索    │  ripgrep (内嵌)                            │
│  构建工具    │  Bun bundler + dead-code elimination      │
└─────────────────────────────────────────────────────────┘
```

**核心技术选型说明：**

| 技术 | 选型理由 |
|------|---------|
| **Bun** | 极快的启动速度，单文件打包，原生 TS 支持 |
| **React + Ink** | 用 React 组件模型构建终端 UI，复用前端开发范式 |
| **Commander.js** | 成熟的 CLI 参数解析器，支持子命令 |
| **Zod v4** | 运行时类型校验，与 TypeScript 类型系统深度集成 |
| **ripgrep** | 极快的代码搜索，支持 .gitignore 过滤 |

## 3. 启动流程

应用的启动是一个精心优化的流程，通过**并行预取**减少冷启动时间：

```
┌──────────────────────────────────────────────────────────────────┐
│                        启动流程 (main.tsx)                        │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │

│  ┌────────────────────┐  ┌────────────────────┐                  │

│  ┌────────────────────┐                                          │
│  │startKeychainPrefetch│                                         │
│  │ 预取 OAuth 密钥     │  ← macOS 钥匙串读取约 65ms               │
│  └────────────────────┘                                          │
│         │                                                        │
│         │ (并行执行)                                               │
│         ▼                                                        │
│  ② 模块加载阶段 (~135ms)                                          │
│  ┌────────────────────────────────────────────┐                  │
│  │ import 重量级模块 (React, Ink, 工具等)       │                  │
│  └────────────────────────────────────────────┘                  │
│         │                                                        │
│         ▼                                                        │

│  ┌────────────────────────────────────────────┐                  │

│  └────────────────────────────────────────────┘                  │
│         │                                                        │

│  ④ 运行模式分发                                                    │
│  ┌──────────┬──────────┬──────────┬────────────┐                 │
│  │ 交互式    │ 非交互式  │ Bridge   │ Daemon     │                 │
│  │ REPL 模式 │ SDK 模式  │ IDE 模式 │ 远程模式    │                 │

│                                                                  │

```

**运行模式说明：**

| 模式 | 触发条件 | 说明 |
|------|---------|------|
| **交互式 REPL** | 默认 | 完整 Ink TUI，支持消息历史、权限提示、工具进度 |
| **非交互式** | `--non-interactive` | JSON 格式输出，无输入提示 |
| **SDK/Print** | `--sdk` | 无 Ink UI，原始结构化输出 |
| **Bridge** | `BRIDGE_MODE` | IDE 扩展控制模式 |
| **Daemon** | `DAEMON` | 后台远程服务模式 |

## 6. 工具系统 (Tool System)

工具系统是 Claude Code 与文件系统、终端、外部服务交互的核心能力。

### 6.1 工具接口定义

```typescript
interface Tool {
  name: string                    // 工具名称
  aliases?: string[]              // 别名
  description: string             // 工具描述（提供给模型）
  inputSchema: ToolInputJSONSchema // 输入参数 JSON Schema

  // 核心方法
  execute(input, context): Promise<ToolResult>

  // 权限控制
  canUseTool?(context): boolean | Promise<boolean>

  // UI 进度回调
  setToolJSX?: SetToolJSXFn

  // 是否启用
  isEnabled?(): boolean
}
```
