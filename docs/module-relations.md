## 2. 数据流向图——一次完整的用户交互

当你输入一句话，数据在各模块之间的流动过程：

```
                          用户: "帮我写个排序函数"
                                    │
                                    ▼
┌─ STEP 1: 接收输入 ─────────────────────────────────────────────────┐
│                                                                    │
│  UI 层 (PromptInput)                                               │
│  │                                                                 │
│  │  接收键盘输入 → 检查是否是斜杠命令 → 不是 → 作为普通消息处理      │
│  │                                                                 │
│  └──→ 交给 QueryEngine                                             │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─ STEP 2: 构建上下文 ───────────────────────────────────────────────┐
│                                                                    │
│  QueryEngine                                                       │
│  │                                                                 │
│  ├──→ 记忆系统: 加载 MEMORY.md 中的记忆文件                         │
│  │    返回: "用户是后端工程师，偏好简洁风格"                          │
│  │                                                                 │
│  ├──→ 配置系统: 读取 CLAUDE.md + settings.json                     │
│  │    返回: 项目规则、权限设置、模型选择                              │
│  │                                                                 │
│  ├──→ 状态管理: 获取当前状态                                        │
│  │    返回: git 分支、已连接的 MCP、Token 用量                      │
│  │                                                                 │
│  └──→ 系统提示词构建: 组装完整提示词                                 │
│       返回: "你是 Claude Code... + 项目规则 + 记忆 + git 状态"      │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─ STEP 3: 调用 API ─────────────────────────────────────────────────┐
│                                                                    │
│  query() 管道                                                      │
│  │                                                                 │
│  └──→ Claude API (流式)                                            │
│       │                                                            │
│       ├── 文字: "好的，我来写一个..." → 直接显示给用户                │
│       │                                                            │
│       └── 工具调用: "FileWrite: src/sort.ts"                       │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─ STEP 4: 权限检查 + 工具执行 ──────────────────────────────────────┐
│                                                                    │
│  权限系统                                                          │
│  │                                                                 │
│  ├──→ 检查 Hook 规则: 没有匹配的自动规则                            │
│  ├──→ 检查配置规则: FileWrite 不在 alwaysAllow 里                  │
│  └──→ 询问用户:                                                    │
│       ┌────────────────────────────────────┐                       │
│       │ Claude 想要创建文件 src/sort.ts    │                       │
│       │ [✅ 允许]  [❌ 拒绝]               │                       │
│       └────────────────────────────────────┘                       │
│       用户点 "允许"                                                 │
│       │                                                            │
│       ▼                                                            │
│  工具系统: FileWriteTool.execute()                                  │
│  │                                                                 │
│  └── 写入 src/sort.ts → 更新文件缓存 → 更新状态                    │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
                                    │
                                    ▼
┌─ STEP 5: 继续对话 ────────────────────────────────────────────────┐
│                                                                    │
│  工具结果返回给 API → AI 看到结果                                   │
│  │                                                                 │
│  ├── AI 可能继续调用工具 (循环回 STEP 4)                            │
│  │                                                                 │
│  └── AI 回复文字: "排序函数已创建在 src/sort.ts"                    │
│      │                                                             │
│      ▼                                                             │
│  遥测分析: 记录这轮对话的 Token 用量、工具调用次数                     │
│  状态管理: 更新成本统计                                              │
│  UI 层: 渲染最终回复                                                │
│                                                                    │
└────────────────────────────────────────────────────────────────────┘
```

## 6. 模块通信方式

不同模块之间用不同方式通信：

```
┌────────────── 模块间通信方式 ──────────────────────┐
│                                                    │
│  📞 直接函数调用 (最常见)                            │
│  ├── QueryEngine → query()                         │
│  ├── query() → StreamingToolExecutor               │
│  ├── 工具系统 → 权限系统                             │
│  └── 命令系统 → 技能系统                             │
│                                                    │
│  📬 状态订阅 (响应式)                                │
│  ├── UI 层 订阅 AppState 变更                       │
│  ├── 配置变更 → 触发多个模块刷新                     │
│  └── MCP 连接状态 → UI 更新                         │
│                                                    │
│  🔄 AsyncGenerator (流式)                           │
│  ├── query() 是 AsyncGenerator                     │
│  ├── API 响应流式传输                               │
│  └── 工具执行结果流式返回                            │
│                                                    │
│  🌐 HTTP Polling (IDE Bridge)                       │
│  ├── IDE 扩展 ←→ Bridge 主循环                      │
│  └── 中转权限请求和执行结果                          │
│                                                    │
│  📡 进程间通信 (多代理 + MCP)                        │
│  ├── Coordinator ←→ Worker 子进程                   │
│  ├── MCP stdio 客户端 ←→ MCP 子进程                 │
│  └── MCP HTTP/SSE/WS 客户端 ←→ 远程服务器           │
│                                                    │
└────────────────────────────────────────────────────┘
```

## 7. 为什么这样设计？设计原则总结

```
┌──────────────────── 设计原则 ──────────────────────┐
│                                                    │
│  1️⃣  安全第一                                      │
│     • 权限系统作为独立层，所有工具调用必经             │
│     • 多来源权限规则，组织策略不可覆盖                │
│     • 敏感操作需用户确认                             │
│                                                    │
│  2️⃣  可扩展性                                      │
│     • MCP 协议让外部工具无限扩展                     │
│     • 插件系统让社区贡献功能                         │
│     • 技能系统让提示词模板可复用                      │
│                                                    │
│  3️⃣  性能优化                                      │
│     • 并行预取减少启动时间                           │
│     • 工具并行执行提高效率                           │
│     • 缓存（文件、配置、上下文）减少重复计算          │
│     • Feature Flag 编译时删除未用代码                │
│                                                    │
│  4️⃣  单一职责                                      │
│     • 每个模块只做一件事                             │
│     • QueryEngine 不直接操作文件（交给工具）          │
│     • 权限系统不直接执行工具（只做决策）               │
│                                                    │
│  5️⃣  用户体验                                      │
│     • 多种运行模式（终端 / IDE / SDK）               │
│     • 可定制的键绑定和 Vim 模式                      │
│     • 记忆系统让跨会话体验连贯                       │
│                                                    │
│  6️⃣  会话隔离                                      │
│     • 每个对话一个 QueryEngine 实例                  │
│     • 子代理有独立上下文                             │
│     • 权限检查原子化（ResolveOnce 防竞态）           │
│                                                    │
└────────────────────────────────────────────────────┘
```

| ⑥ 权限系统 | 配置系统、状态管理 | 工具系统、查询管道 |
